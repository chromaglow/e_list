## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-A | Tampering (XSS) | Browse page SSR — rendering DB listing content | mitigate | React escapes all content in JSX text positions; ListingCard has no dangerouslySetInnerHTML; all user content goes through JSX interpolation |
| T-02-05-B | Information Disclosure | edit_token rendered in browse page | mitigate | BrowsePage passes the Listing object to ListingCard; ListingCard does not render listing.edit_token — confirmed in Plan 02-03 task acceptance criteria |
| T-02-05-C | Spoofing | AppHeader href contains invite token visible in page source | accept | The invite token is already in the URL bar — exposing it in the href is not additional disclosure; the invite gate relies on the token being shared with trusted friends |
| T-02-05-D | Tampering | getActiveListings() query injection | mitigate | Drizzle parameterized query — no user input goes into the SELECT query; query is fixed (filter: status='active', order: created_at DESC) |
| T-02-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
