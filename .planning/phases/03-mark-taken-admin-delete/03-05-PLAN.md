## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server Component → Client island | isAdmin computed server-side; passed as prop; client island only controls UI visibility |
| BrowsePage → DB | Server reads admin_session cookie then queries DB; no client trust required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-18 | Elevation of Privilege | isAdmin prop — client manipulation | mitigate | isAdmin is server-computed from verifyAdminSession; DELETE route handler re-verifies JWT independently (defense in depth) |
| T-03-19 | Information Disclosure | BrowsePage renders taken listings to all users in 'taken' tab | accept | Taken listings are intentionally public (status is non-sensitive); listing content already visible to all invite-link holders |
| T-03-20 | Information Disclosure | RegenInviteForm displays new invite URL | accept | Only rendered after successful admin-authenticated POST; admin is the intended recipient |
| T-03-21 | Tampering | filter='deleted' in URL param | mitigate | validFilter ternary only accepts 'taken' and 'all'; any other value (including 'deleted') defaults to 'active'; getListingsByFilter never queries status='deleted' |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase |
</threat_model>
