    - State: `const [isPending, setIsPending] = useState(false)`

    - Early return: `if (!hasToken) return null` — renders nothing on server and before hydration
    - `handleClick` async function: reads `localStorage.getItem('edit_token_' + listingId)`, returns early if null, sets `isPending(true)`, fetches `/${token}/api/listings/${listingId}` with method PATCH, Content-Type application/json, body `JSON.stringify({ editToken })`. On `res.ok`: calls `router.refresh()` (re-fetches server component — card disappears from Active tab per D-04). Sets `isPending(false)` in finally block.
    - Returns: `<Button variant="outline" size="sm" onClick={handleClick} disabled={isPending}>{isPending ? 'Marking…' : 'Mark as taken'}</Button>`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client component → Route Handler | All secrets (editToken) travel in POST/PATCH body over HTTPS, never in URL |
| localStorage → PATCH body | editToken is read from localStorage and sent to server; server does the actual verification |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | MarkTakenButton — editToken in request body | mitigate | editToken travels in JSON body (never URL); server verifies with timingSafeEqual (Plan 03) |
| T-03-05 | Elevation of Privilege | AdminDeleteButton — isAdmin gating | mitigate | isAdmin prop is server-rendered (D-11); AdminDeleteButton only controls UI visibility; actual auth check is in DELETE route handler (Plan 03) |
| T-03-06 | Spoofing | RegenInviteForm — new URL display | accept | Only shows URL after successful admin-authenticated POST; no secret is exposed beyond what admin already knows |
| T-03-07 | Information Disclosure | FilterTabs — ?filter= URL param | accept | Filter value is non-sensitive (active/taken/all); public URL is acceptable |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; lucide-react already installed per RESEARCH.md standard stack |
</threat_model>
