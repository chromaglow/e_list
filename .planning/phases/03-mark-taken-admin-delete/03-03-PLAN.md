    PATCH handler:
    - Zod schema: `{ editToken: z.string().min(1).max(500) }`
    - JSON parse fails → 400
    - Zod parse fails → 400
    - DB lookup: SELECT edit_token, status FROM listings WHERE id = [id param]
    - Listing not found → 404
    - Listing status !== 'active' → 404 (listing already taken or deleted)
    - timingSafeEqual comparison of supplied editToken vs DB edit_token
    - Token mismatch → 403
    - Token match → calls markListingTaken(id) → returns { ok: true } 200

    DELETE handler:
    - No request body needed
    - Reads admin_session cookie with `(await cookies()).get('admin_session')?.value`
    - Calls verifyAdminSession(sessionCookie)
    - No valid session → 403
    - Valid session → calls deleteListingAdmin(id) → returns { ok: true } 200
  </behavior>
  <action>
    Create app/[token]/api/listings/[id]/route.ts (new file — directory [id] does not yet exist).

    Imports (in order):
    - `import { z } from 'zod'`
    - `import { timingSafeEqual } from 'node:crypto'`
    - `import { cookies } from 'next/headers'`
    - `import { db } from '@/lib/db'`
    - `import { listings } from '@/lib/schema'`
    - `import { eq } from 'drizzle-orm'`
    - `import { verifyAdminSession } from '@/lib/session'`
    - `import { markListingTaken, deleteListingAdmin } from '@/lib/listing-service'`

    Define Zod schema: `const MarkTakenSchema = z.object({ editToken: z.string().min(1).max(500) })`

    Export `async function PATCH(request: Request, { params }: { params: Promise<{ token: string; id: string }> })`:
    1. `const { id } = await params`
    2. JSON parse with try/catch → 400 on failure
    3. `MarkTakenSchema.safeParse(json)` → 400 on failure
    4. DB select: `db.select({ edit_token: listings.edit_token, status: listings.status }).from(listings).where(eq(listings.id, id)).get()`
    5. If no row or `row.status !== 'active'` → `Response.json({ error: 'Not found' }, { status: 404 })`
    6. `timingSafeEqual` comparison — copy the exact try/catch pattern from proxy.ts lines 31-39, comparing `Buffer.from(parsed.data.editToken)` vs `Buffer.from(row.edit_token)`. Note: edit tokens are UUID format (not hex), so use `Buffer.from(str)` not `Buffer.from(str, 'hex')`.
    7. If tokenMatch is false → `Response.json({ error: 'Forbidden' }, { status: 403 })`
    8. `await markListingTaken(id)`
    9. `return Response.json({ ok: true })`

    Export `async function DELETE(request: Request, { params }: { params: Promise<{ token: string; id: string }> })`:
    1. `const { id } = await params`
    2. `const sessionCookie = (await cookies()).get('admin_session')?.value`
    3. `const session = await verifyAdminSession(sessionCookie)`
    4. If no session → `Response.json({ error: 'Forbidden' }, { status: 403 })`
    5. `await deleteListingAdmin(id)`
    6. `return Response.json({ ok: true })`

    - `import { randomBytes } from 'node:crypto'`
    - `import { cookies } from 'next/headers'`
    - `import { verifyAdminSession } from '@/lib/session'`
    - `import { setSetting } from '@/lib/settings-service'`

    Export `async function POST()` (no params needed — admin gate is in middleware and verified here):
    1. `const sessionCookie = (await cookies()).get('admin_session')?.value`
    2. `const session = await verifyAdminSession(sessionCookie)`
    3. If no session → `return Response.json({ error: 'Forbidden' }, { status: 403 })`
    4. `const newToken = randomBytes(32).toString('hex')` — produces 64 hex chars (32 bytes × 2 hex digits)
    5. `await setSetting('invite_token', newToken)` — upserts the DB row, invalidating the old token
    6. `return Response.json({ newToken })`

    Note: The middleware cache in proxy.ts has up to 60s TTL. After this write, the new token will be active for all requests once the cache expires. The RegenInviteForm client already shows the new URL immediately from the response, so users know to use the new URL right away even if there's a brief overlap window.
