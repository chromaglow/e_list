<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server -> Set-Cookie header | Cookie flags (HttpOnly, Secure, SameSite=Strict) prevent JS access, plaintext transport, and cross-site form submission |
| ADMIN_PASSWORD_HASH env var -> bcrypt.compare (Plan 03b) | Trusted hash; integrity assumed; rotation requires redeploy |
| createAdminSession -> JWT signing | jose HS256 over SESSION_SECRET (declared by Plan 01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Spoofing | Weak password hashing | mitigate | scripts/gen-hash.js hardcodes cost 12; refuses CLI flag to lower it; output regex-validated by Plan 01's env loader |
| T-03-04 | Tampering | JWT algorithm confusion at signing time | mitigate | createAdminSession uses SignJWT().setProtectedHeader({ alg: 'HS256' }) explicitly; verifyAdminSession (Plan 01) already locked to algorithms: ['HS256'] |
| T-03-06 | Tampering | CSRF on admin actions | mitigate | sameSite: 'strict' on admin_session cookie (RESEARCH.md Pattern 2 / Pitfall 9); blocks any cross-site POST from carrying the cookie |
| T-03-07 | Spoofing | Stolen cookie replay | mitigate | httpOnly: true blocks JS access; secure: true in production blocks HTTP transmission; 4h expiry limits replay window |
| T-03-09 | Denial of Service | In-memory rate limit ineffective across Vercel cold starts | accept | RESEARCH.md A1 — for a single-admin hobby deploy, cold-start instance isolation is acceptable; upgrade path is Upstash Redis if needed |
| T-03-SC | Tampering | bcryptjs / zod / jose install | accept | All verified in RESEARCH.md Package Legitimacy Audit (npm registry, 2026-05-17) |
</threat_model>
