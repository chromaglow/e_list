<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application -> Turso | Authenticated via TURSO_AUTH_TOKEN (JWT); URL+token must NEVER appear in client bundles |
| Migration runner -> Turso | Same auth surface, but with DDL privileges; tokens are write-scoped |
| Schema definition -> DB | Status enum, NOT NULL constraints, and default values are the runtime guarantee ŌĆö application code MUST NOT bypass via raw SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | lib/db.ts client | mitigate | `import 'server-only'` at top of lib/db.ts and lib/schema.ts so any accidental client import fails at build time |
| T-02-02 | Tampering | Drizzle status column | mitigate | Use Drizzle `text({ enum: [...] })` typing AND a CHECK constraint in the generated SQL; reject `'sold'`, `'pending'`, etc. at the DB layer (defense in depth) |
| T-02-03 | Tampering | Migration file drift between local and prod | mitigate | D-07 ŌĆö only `drizzle-kit generate` + `drizzle-kit migrate` used; `push` is documented as forbidden in package.json scripts (no `db:push` script defined) |
| T-02-04 | Information Disclosure | TURSO_AUTH_TOKEN leakage | accept | Token is required by drizzle-kit at build time; stored in Vercel encrypted env vars in Plan 05; rotation if leaked is one-click in Turso dashboard |
| T-02-05 | Denial of Service | Schema with no row cap | accept | Turso free tier (5 GB, 500M reads/month) and friend-group scale (5-20 users) make capacity exhaustion implausible; Pitfall 13 (auto-archive) is Phase 4 |
| T-02-06 | Repudiation | Soft-delete + timestamps | mitigate | created_at/updated_at/taken_at/deleted_at give an audit trail for admin actions and status transitions per CLAUDE.md schema notes |
| T-02-SC | Tampering | drizzle-orm, drizzle-kit, @libsql/client install | accept | Verified in RESEARCH.md Package Legitimacy Audit (npm registry, 2026-05-17); slopcheck false-SLOP verdicts on drizzle packages explicitly documented as wrong-ecosystem |
</threat_model>
