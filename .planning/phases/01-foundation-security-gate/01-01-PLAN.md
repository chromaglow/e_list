<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> Vercel Edge | Anonymous traffic; only invite-URL holders should pass |
| URL path segment -> proxy.ts | Untrusted first segment compared to INVITE_TOKEN env secret |
| Cookie header -> proxy.ts | Untrusted `admin_session` JWT verified against SESSION_SECRET |
| process.env -> application | Trusted but must exist; missing values must crash boot, not silently fail-open |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | proxy.ts invite check | mitigate | Constant-time string comparison via `timingSafeEqual` of Buffer-from-hex inputs; reject if length != 64 hex chars before compare |
| T-01-02 | Information Disclosure | proxy.ts invite gate | mitigate | Use `NextResponse.rewrite(new URL('/not-found', request.url))` (per RESEARCH.md Pitfall 3) so URL is preserved but 404 returned; do NOT redirect to a path that echoes the bad token |
| T-01-03 | Information Disclosure | INVITE_TOKEN value | accept | URL-path tokens unavoidably appear in browser history and access logs; token-rotation UI is Phase 3 (ADMN-04); document risk in `.env.example` |
| T-01-04 | Tampering | JWT cookie verification | mitigate | `jwtVerify(token, key, { algorithms: ['HS256'] })` — explicit algorithm list defeats `alg: none` and algorithm confusion attacks |
| T-01-05 | Elevation of Privilege | Admin path detection in proxy | mitigate | Skip admin-JWT check when path is `/{token}/admin/login` exactly (per CONTEXT.md D-01 + RESEARCH.md Pitfall 4) so login page itself is reachable; only require JWT for `/{token}/admin` and deeper non-login segments |
| T-01-06 | Denial of Service | proxy.ts matcher | accept | Vercel Edge has built-in rate limits; in-memory limiter on login is Plan 03; broad path matcher is required by CONTEXT.md D-03 ("every server request") |
| T-01-07 | Spoofing | Missing SESSION_SECRET / INVITE_TOKEN at boot | mitigate | `lib/env.ts` throws on first access if value is unset; prevents fail-open where empty string matches empty path segment |
| T-01-SC | Tampering | npm install of next/react/jose/etc | accept | All packages verified in RESEARCH.md `## Package Legitimacy Audit` against npm registry on 2026-05-17 with no [ASSUMED]/[SUS] flags; checkpoint task gates the install step but no per-package human verification required |
</threat_model>
